# Security Policy

## Supported version

Security fixes are applied to the latest published release.

## Reporting a vulnerability

Do not open a public issue for vulnerabilities that could expose credentials, private repositories, unsafe filesystem behavior, command execution, or authority-boundary bypasses.

Use [GitHub Security Advisories](https://github.com/emanueledenaro/ultracode/security/advisories/new) to report the issue privately. Include the affected version, reproduction steps, impact, and any suggested mitigation. Do not include real secrets or unrelated private data.

You should receive an acknowledgement within seven days. A fix, advisory, and release will be coordinated according to severity and reproducibility.
